# 寻风集 APP 产品需求文档

> [!abstract] 产品概述
> 寻风集 APP 旨在为户外爱好者提供一站式解决方案，重点解决户外专用车辆的租赁、户外装备的获取、活动参与及社交购物需求，并通过 AI 技术实现个性化内容推荐。

## 1. 功能总览

| 模块 | 核心内容 |
| --- | --- |
| 租赁服务系统 | 车辆租赁、物品租赁、押金、保险、会员权益 |
| 活动管理系统 | 路线体系、活动发布、报名审核、通知订阅、保险与推荐 |
| 内容（社区）服务和管理系统 | 直播、语音交流、内容发布、互动与后台审核 |
| 会员管理系统 | 注册登录、实名认证、资产体系、优惠卷体系 |
| 智能推荐系统 | 基于行为与偏好的个性化推荐 |
| 购物平台 | 官方商城、二手循环集市 |
| 支撑服务系统 | 支付、保险、发票、客服、问卷、订阅、公众号、埋点、文件存储 |

## 2. 核心功能模块需求

### 2.1 租赁服务系统

> [!info] 模块说明
> 这是平台的核心业务模块，包含完整的租赁闭环。

#### 2.1.1 车辆租赁

- 管理员在后台管理车辆库存和维护车辆信息，用户可以在平台上查看车辆信息并进行租赁。
- 提供灵活的租赁服务，如多城市取车、送车上门、异地还车和灵活的租赁时间。
- 多样的增值服务，为高等级会员提供更多功能和服务，如优先服务、会员折扣等。
- 只有注册并实名认证后的用户才能进行车辆租赁。
- 一般用户需要支付押金，押金可以在租赁结束后返回。
- 提供多种保险选项，满足不同风险偏好的用户。

#### 2.1.2 物品租赁

- 管理员在后台管理物品库存和维护物品信息，用户可以在平台上查看物品信息并进行租赁。
- 只有注册并实名认证后的用户才能进行物品租赁。
- 一般用户需要支付押金，押金可以在租赁结束后返回。

### 2.2 活动管理系统

> [!info] 模块说明
> 平台会不定期组织户外活动，用户可以在平台上查看活动信息并进行报名。活动也可以由注册的组织者在平台上创建和管理。

#### 2.2.1 路线体系

- 活动以“路线”为中心，每个活动都有一个具体的路线，用户可以在平台上查看路线地图并进行报名。
- 路线以路书的形式展示，包含地点、详细路线、距离、时间、难度、后勤保障等信息。
- 每条路线有评分和星级。
- 用户可以自己创建路线，路线需要经过平台审核，审核通过后才可以在平台上展示。

#### 2.2.2 活动发布与报名

- 组织者用户可以在平台上发布组织活动，开启活动招募，活动可以设置互动的报名费。
- 活动可以设置用户报名时需要提供的个人信息，如姓名、手机号、身份证等。
- 只有注册并实名认证后的用户才能报名活动，且只能是本人报名。
- 在活动开始前，用户可以取消报名；取消后如果活动仍可报名，将可以重新报名。
- 如果活动需要支付费用，用户需要在报名时支付费用；如果超时未支付，平台会自动取消用户的报名。
- 用户成功取消报名后，平台会自动退回用户支付的费用。
- 活动中可能会为参与者购买保险（企业用户）。

#### 2.2.3 活动管理与通知

- 组织者用户和平台管理者可以在手机或平台上查看活动报名情况，并可以对报名用户进行管理（如同意报名、拒绝报名、邀请好友报名等）。
- 用户通知包括：
  - 报名状态变更：用户报名成功或失败后，平台会发送确认通知到用户的手机号。
  - 活动变更：如果活动时间、地点、活动介绍、活动规则有变更，会发送通知到所有报名用户。
- 平台管理者可以对活动进行审核和管理，并发送审核结果通知到组织者用户。
- 用户可以订阅新活动活动信息，订阅时可以指定活动发起人、活动地点、活动时间等。

#### 2.2.4 智能匹配与招募

- 可以根据用户的要求和用户的侧写数据为用户推荐活动路线以及装备。
- 如推荐的路线当前没有招募中活动，用户可以在平台上呼唤领队和同行者，由领队开启活动招募。

### 2.3 内容（社区）服务和管理系统

#### 2.3.1 内容生产与互动

- 用户在参加活动时，可以视频直播活动，直播中用户可以实现互动。
- 为户外活动提供语音交流集群服务，用户可以在集群中与其他用户进行实时语音交流。
- 用户可以在平台上发布和分享自己的户外经历、活动记录、装备使用经验等。
- 用户可以对其他用户发布的内容进行点赞、评论、收藏、分享、私信或通知等操作。

#### 2.3.2 内容分发与管理

- 平台会根据用户的活动记录和分享内容，推荐相关的活动和装备。
- 管理员可以在后台管理用户发布的内容，包括审核、删除、修改等操作。

### 2.4 会员管理系统

#### 2.4.1 用户注册登录与账号管理

- 用户分为企业用户和个人用户，个人用户分为普通用户和组织者用户。
- 用户账号创建、登录、密码找回：
  - 用户注册使用手机号或邮箱方式。
  - 手机号注册可以通过短信验证码验证。
  - 邮箱注册可以通过邮箱链接验证。
- 用户账号设置、个人信息维护（昵称、头像、简介、手机号、邮箱）。
- 用户账号实名认证：
  - 企业用户需要提供企业资质（如营业执照、组织机构代码证等）。
  - 个人用户需要提供个人资质（如身份证、驾驶证等）。

#### 2.4.2 用户资产管理

- 用户资产管理包含余额、积分、优惠卷、等级。
- 普通用户：通过充值获得余额和积分，余额可用于平台消费。
- 组织者用户：通过发布组织活动获得积分，积分可用于升级组织者等级。
- 所有用户都可以通过发布内容获并达到一定要求后获得积分和优惠卷。

#### 2.4.3 优惠卷管理

- 用户可以通过多种渠道获取优惠卷，如充值、发布组织活动、发布内容等。
- 优惠卷有多个种类，如现金抵扣类型、消费折扣类型等。
- 优惠卷可以设定使用条件，例如指定时间范围、指定活动范围、指定商品、指定使用者等级和积分最低要求等。
- 单次消费只能使用一张优惠卷。

### 2.5 智能推荐系统

- 基于用户行为和偏好，推荐相关的活动、装备、路线及直播内容等。
- 推荐算法会根据用户的活动记录、分享内容、购买记录等进行分析和学习，提供个性化的推荐。

### 2.6 购物平台

- 商城：参考主流电商模式（如德物）建设官方商城。
- 二手循环集市：参考二手交易平台（如闲鱼）建设，促进户外装备的二次流通。

### 2.7 支撑服务系统

#### 2.7.1 支付系统

- 在线支付，支持支付宝、微信支付 + 积分 + 优惠卷。

#### 2.7.2 保险系统

- 管理员在后台管理保险产品库存，用户可以在平台上查看保险产品信息并进行购买。
- 保险购买需要用户提供个人信息（如姓名、手机号、身份证等），并需要进行审核。

#### 2.7.3 发票系统

- 用户可以在订单支付后生成电子发票。
- 发票包含订单详情、支付金额、发票号等信息。
- 用户可以在平台上查看自己的发票记录。

#### 2.7.4 客服系统

- 提供 24 小时在线客服，用户可以通过客服系统与客服进行咨询和问题解决。
- 客服可以在客服系统中查看用户的问题和咨询记录，并可以与用户进行实时互动。

#### 2.7.5 问卷系统

- 用户可以在平台上填写问卷，收集用户反馈和建议。
- 问卷可以设定使用条件，例如指定时间范围、指定活动范围等。

#### 2.7.6 订阅系统

- 用户可以在平台上订阅指定的活动或装备，收到相关更新和通知。
- 订阅可以设定使用条件，例如指定时间范围、指定活动范围等。

#### 2.7.7 公众号

- 平台会创建一个官方公众号，用户可以关注公众号并接收相关资讯和活动通知。

#### 2.7.8 埋点系统

- 平台会在用户使用 APP 时，记录用户的操作行为（如点击、滑动、搜索等）。
- 埋点数据会被分析和统计，用于优化 APP 功能和用户体验。

#### 2.7.9 文件存储系统

- 平台会使用云存储服务（如 Aliyun OSS）存储用户上传的图片、视频、文件等。
- 存储系统会提供高可用性、可扩展性和安全性。

## 3. 非功能需求

### 3.1 性能

- 客户端关键页面首屏 ≤ 2s（4G 网络），交互响应 ≤ 100ms。
- 推荐流滚动无明显卡顿；图片视频懒加载与分片传输。
- 后端 P99 接口 ≤ 300ms；高峰并发 1–5 万 QPS 可横向扩展。

### 3.2 兼容性

- 支持 iOS 14+ / Android 8+。
- 适配主流机型。
- 支持弱网容错与断点续传。

### 3.3 安全与合规

- 支持 OAuth2/JWT、HTTPS 全链路。
- 满足隐私合规要求（数据最小化、可删除）。
- 支持审计日志、操作留痕、防刷与限流。
- 满足版权合规与未成年人保护要求。

### 3.4 可扩展性与可维护性

- 推荐与审核能力可独立扩展。
- 通过缓存与消息队列解耦系统能力。
- 建立统一日志、指标、告警体系。
- 支持灰度发布、快速回滚、在线开关与阈值控制。

### 3.5 可观测性

- 建立埋点规范与校验机制。
- 支持链路追踪与可用性 SLA 监控。
- 建立质量看板与告警分级能力。
